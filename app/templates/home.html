Introduction

This Project is from Data Science Trainee program in Jakarta Smart City. In this project we are trying to implement Few Shot Learning method in order to categorize CRM (Cepat Respon Masyarakat) image report from JAKI (Jakarta Kini) app. As Trainee I was mentored by these great people:

Common practice for machine learning applications is to feed as much data as the model can take. This is because in most machine learning applications feeding more data enables the model to predict better. However, few-shot learning aims to build accurate machine learning models with less training data (aimultiple).

Few-Shot Learning is an example of meta-learning, where a learner is trained on several related tasks, during the meta-training phase, so that it can generalize well to unseen (but related) tasks with just few examples. An effective approach to the Few-Shot Learning problem is to learn a common representation for various tasks (paperswithcode).

Datasets

The datasets we use in this project is from CRM report in JAKI aplication, after cleaning process we get 13 class target which have distinction from each other.

In Training process we use 10 classes and 3 classes for evaluation which are the lowest number of samples; Communication Netwok, Clean Water Network and Sidewalk. Training process use 5-Way 5-Shot for each task and trained for 2000 tasks.

Methodology

We training two model architectures base on Relation Network (Sung et al., 2018) and Prototypical Network (Snell et al., 2017) for feature extractor we use three different pretrained models on ImageNet VGG16, Resnet50 and MobileNetV3.

Evaluation is done by seeing accuracy on three unseen class; Communication Netwok, Clean Water Network and Sidewalk if the models don't learn we look back the dataset and clean it again.
